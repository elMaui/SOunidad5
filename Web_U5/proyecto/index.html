Bienvenid@ a la Unidad V

Un sistema operativo no solo gestiona recursos computacionales esenciales como procesos, hilos, archivos y espacios de direcciones; también desempeña un papel clave en la administración de todos los periféricos de entrada y salida conectados al sistema. Esta responsabilidad implica coordinar la comunicación con los dispositivos, responder adecuadamente a las interrupciones generadas por ellos, y resolver fallos que puedan surgir durante su operación. Para facilitar el uso de los dispositivos, el sistema operativo ofrece una capa de abstracción que actúa como intermediaria entre el hardware y el software. Esta interfaz debe ser lo más uniforme posible para todos los tipos de dispositivos, permitiendo así que los programas puedan interactuar con ellos sin preocuparse por las diferencias específicas del hardware. Este enfoque de "independencia de dispositivo" mejora la portabilidad y simplifica el desarrollo de software. La gestión de entrada/salida representa una parte significativa del código del sistema operativo. Debido a su complejidad y su importancia para la estabilidad y eficiencia del sistema, la administración de E/S se divide en varios niveles funcionales, cada uno con responsabilidades concretas. Estos niveles cooperan para manejar solicitudes de E/S de manera ordenada, desde las llamadas del usuario hasta la ejecución física de la operación en el dispositivo. En el desarrollo de este tema, primero se exploran los fundamentos del hardware de entrada/salida: cómo se comunican los dispositivos con el procesador, los mecanismos de interrupciones, los registros de control, y las técnicas de transferencia de datos como el acceso directo a memoria (DMA). Luego se estudia el software de E/S, estructurado en capas como los controladores de dispositivos, las rutinas de manejo de interrupciones, y las interfaces de alto nivel. Posteriormente, se profundiza en el funcionamiento de dispositivos específicos, como discos duros, relojes del sistema, teclados y monitores, analizando tanto sus características físicas como los métodos empleados por el sistema operativo para controlarlos eficientemente. Finalmente, se aborda un tema cada vez más relevante: la gestión del consumo energético en sistemas modernos. Esto incluye estrategias para reducir el uso de energía sin comprometer el rendimiento, como la suspensión selectiva de dispositivos inactivos y el ajuste dinámico del rendimiento del procesador.
